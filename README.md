# pg-propre.el

Format and lint PostgreSQL SQL inside Emacs using
[`pg_propre`](https://gitlab.com/madtibo/pg_propre), a fast Rust SQL formatter
and linter built on PostgreSQL's own parser.

This package provides:

- **Formatting** — `pg-propre-format-buffer` and `pg-propre-format-region`
  commands plus a `pg-propre-format-on-save-mode` minor mode, generated by the
  [reformatter](https://github.com/purcell/emacs-reformatter) library.
- **Linting** — `pg-propre-flymake`, a [Flymake](https://www.gnu.org/software/emacs/manual/html_node/flymake/)
  backend driving `pg_propre --lint`, for live diagnostics as you edit.

## Requirements

- Emacs 26.1 or newer (for the Flymake backend API).
- The [`reformatter`](https://melpa.org/#/reformatter) package (from MELPA).
- The `pg_propre` executable on your `PATH`. Install it with
  `cargo install pg_propre`, or grab a prebuilt binary from the
  [Releases page](https://gitlab.com/madtibo/pg_propre/-/releases).

## Installation

Ensure `pg-propre.el` is on your `load-path` (and `reformatter` is installed),
then:

```elisp
(require 'pg-propre)
```

## Usage

Open a SQL buffer and:

- `M-x pg-propre-format-buffer` — reformat the whole buffer.
- `M-x pg-propre-format-region` — reformat the active region.
- `M-x pg-propre-format-on-save-mode` — reformat automatically on save.
- `M-x pg-propre-setup-flymake` then `M-x flymake-mode` — show lint diagnostics.

A typical configuration enables everything in `sql-mode`:

```elisp
(add-hook 'sql-mode-hook
          (lambda ()
            (pg-propre-format-on-save-mode)
            (pg-propre-setup-flymake)
            (flymake-mode)))
```

You can also enable on-save formatting per project via `.dir-locals.el`:

```elisp
((sql-mode
   (mode . pg-propre-format-on-save)))
```

## Customisation

`pg_propre` reads its own configuration from `.proprerc` and `.editorconfig`
files, so no arguments are needed by default.

- `pg-propre-command` — name of, or path to, the executable (default
  `"pg_propre"`).
- `pg-propre-args` — extra formatting arguments. For example, to use a style
  preset:

  ```elisp
  (setq pg-propre-args '("--style" "celko"))
  ```

  The lint backend always passes `--lint` and does not use `pg-propre-args`.

## Development

```bash
make compile       # byte-compile (requires reformatter from MELPA)
make package-lint  # run package-lint
make test          # run the ERT suite (needs pg_propre on PATH)
```

Run a single test:

```bash
emacs -Q -batch -l pg-propre.el -l pg-propre-tests.el \
  --eval '(ert-run-tests-batch-and-exit "pg-propre-tests-format-buffer")'
```
